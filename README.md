# E-Auction System Backend

Цифровая платформа для проведения аукционов на Spring Boot.

## Технологический стек
- **Java 21**, **Spring Boot 3.4.2**
- **Spring Security + JWT** (Аутентификация и авторизация)
- **Spring Data JPA + PostgreSQL** (База данных)
- **Liquibase** (Миграции БД)
- **MapStruct & Lombok** (Маппинг и сокращение шаблонного кода)
- **WebSocket** (Обновления ставок в реальном времени)
- **Docker & Docker Compose** (Контейнеризация)
- **Swagger (OpenAPI)** (Документация API)

## Архитектура
Проект следует многослойной архитектуре:
- `web.controller`: REST контроллеры
- `service`: Бизнес-логика и планировщик (Scheduler)
- `repository`: Слой доступа к данным
- `model.entity`: Сущности БД
- `web.dto`: Объекты передачи данных
- `security`: Конфигурация безопасности и JWT

## Как запустить

### 1. С помощью Docker Compose (Рекомендуется)
```bash
docker-compose up --build
```
Это запустит и базу данных PostgreSQL, и само приложение. Приложение будет доступно по адресу `http://localhost:8080`.

### 2. Локальный запуск (Maven)
Сначала запустите только базу данных:
```bash
docker-compose up db -d
```
Затем запустите приложение:
```bash
./mvnw spring-boot:run
```

## API Документация
После запуска документация Swagger доступна по адресу:
`http://localhost:8080/swagger-ui.html`

## Основные функции
- **Регистрация и Логин**: Получение JWT токена.
- **Аукционы**: Создание, просмотр и удаление (с учетом ролей ADMIN/SELLER).
- **Ставки**: Совершение ставок с проверкой цены и времени.
- **Anti-sniping**: Автоматическое продление аукциона на 5 минут, если ставка сделана в конце.
- **Live Bidding**: Подписка на `/topic/auction/{id}` через WebSocket для получения уведомлений о новых ставках.
- **Scheduler**: Автоматическое завершение аукционов и определение победителя каждую минуту.

## Роли пользователей
- `BIDDER`: Может просматривать аукционы и делать ставки.
- `SELLER`: Может создавать свои аукционы.
- `ADMIN`: Полный доступ к управлению системой.
